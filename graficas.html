SUMA

Para diferentes dimensiones de las matrices, se muestra tiempo vs nmero de threads. :

  • El tiempo de rejecucin se reduce conforme se aumenta el nmero de threads.
  • Una almacenamiento por filas o columnas no afecta al rendimiento de la suma de matrices.

PRODUCTO

Para diferentes dimensiones de las matrices, se muestra tiempo vs nmero de threads. Se pueden apreciar los siguientes hechos:

  • El tiempo de rejecucin se reduce conforme se aumenta el nmero de threads.
  • Cuanto mayor es el tamao de las matrices, ms mejora el tiempo de ejecucin al aumentar los threads (con tamao bajo se estanca antes).
  • Una disposicin por columnas es ms conveniente que por filas para la implementacin de la multiplocacin actual. Esto es debido a que en la matriz producto se recorren los elementos por columnas y por tanto es ms fcil trabajar con los bloques de memoria con esa disposicin.
