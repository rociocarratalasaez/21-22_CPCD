SUMA

Para diferentes dimensiones de las matrices, se muestra tiempo vs número de threads. :

  • El tiempo de rejecución se reduce conforme se aumenta el número de threads.
  • Una almacenamiento por filas o columnas no afecta al rendimiento de la suma de matrices.

PRODUCTO

Para diferentes dimensiones de las matrices, se muestra tiempo vs número de threads. Se pueden apreciar los siguientes hechos:

  • El tiempo de rejecución se reduce conforme se aumenta el número de threads.
  • Cuanto mayor es el tamaño de las matrices, más mejora el tiempo de ejecución al aumentar los threads (con tamaño bajo se estanca antes).
  • Una disposición por columnas es más conveniente que por filas para la implementación de la multiplocación actual. Esto es debido a que en la matriz producto se recorren los elementos por columnas y por tanto es más fácil trabajar con los bloques de memoria con esa disposición.
